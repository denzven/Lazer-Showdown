## вњ¦ What Is Lazer Showdown?

**Lazer Showdown** is a two-player tactical board game played on an **8Г—8 grid** filled with fixed diagonal mirrors. One player is the **Attacker**, wielding a laser piece. The other is the **Defender**, protecting high-value point pieces by repositioning them around the board.

A roll of the dice determines your **Action Points**. Use them to move your piece, rotate the laser, and fire вЂ” but every shot bounces off the board's mirrors unpredictably. Mastering the geometry of the beam is the key to victory.

> *"The original name was 'La-Sir-She-Dan' вЂ” Battle of the Land at Ser. The humans couldn't pronounce it. So they called it Lazer Showdown. And somehow, that name ended a war."*

## рџ“‹ The Rules

> *This is a condensed summary. The full official ruleset is inside the game under* **Rules***.*

### рџЏ—пёЏ Structure
```
GAME  в”Ђв†’  2 SETS
SET   в”Ђв†’  3 ROUNDS
ROUND в”Ђв†’  Defender's Roll  +  Attacker's Roll
```

### вљ™пёЏ Setup
1. **Toss** вЂ” Both players roll one die. The higher roller wins and **chooses their role**.
2. **Defender** places **3 Point Pieces** anywhere on the board *except* the four corners.
3. **Attacker** places the **LAZER Piece** on any corner square, facing any direction.

### рџЋІ Your Turn (Rolling & Actions)
- Roll **both dice** вЂ” the total is your **Action Points (AP)**.
- Spend each AP on one of three actions:

| Action | Cost | Who Can Use |
|--------|------|-------------|
| **MOVE** | 1 AP | Attacker moves LAZER 1 square (H/V only). Defender moves a Point Piece 1 square. |
| **ROTATE** | 1 AP | Attacker rotates the LAZER piece clockwise or counter-clockwise. |
| **LAZER PRESS** | 1 AP | Fires the laser. It travels, bounces off mirrors, and captures any Point Piece it hits. |

- Actions can be taken in **any order**.
- Unused AP is **forfeited**.

### рџ’Ґ The Laser
When fired, the laser travels in a straight line until it:
- **Bounces** off a `/` or `\` mirror (90В° reflection).
- **Captures** a Point Piece (piece is removed, points awarded to Attacker).
- **Exits** the board.

Mirror loops are detected and broken automatically to prevent infinite bouncing.

### рџЏ† Scoring & Objectives

| Piece | Points |
|-------|--------|
| Small Piece | 20 pts |
| Medium Piece | 30 pts |
| Large Piece | 50 pts |

- **Attacker** wants to capture all pieces and score as many points as possible.
- **Defender** wants to evade capture, delay the attacker, and survive all 3 rounds.

### вљЎ Challenge Mechanic
If the Attacker captures **all 3 pieces** before the set ends, they may declare a **CHALLENGE**:

1. The Attacker nominates one previously captured piece.
2. A **toss** is rolled (one die per player).
3. **Attacker wins toss** в†’ Defender re-places all pieces; the game continues. If the challenged piece is captured again, its value is **added** to the score.
4. **Attacker loses toss** в†’ The nominated piece's value is **deducted** from the Attacker's score.

Multiple challenges can occur in a single set!

### рџ”„ Between Sets
After Set 1, **roles are swapped**. The former Attacker becomes the Defender, and vice versa. Scores carry over. The player with the **highest total score** after both sets wins. A tie is a **DRAW**.

In **3042 AD**, the **Intergalactic Treaty of Kylos** was signed, ending decades of war. Lazer Showdown was no longer a weapon of intelligence вЂ” it became a bridge between civilizations. Today it is played across the galaxy as a sport of diplomacy.

> *"Tickle its foot. That's all it takes."*

A furry, baseball-sized creature from the **High-Energy Vents of Planet Ser**. It feeds on ambient radiation, making it essentially a living battery. When startled or tickled, it vents excess radiation through a singular ocular lens вЂ” creating a focused beam of pure energy.

In the human version of the game, the **Attacker's Laser Piece** is modeled after the Lzrshuby.

### рџЄЁ The Mirlug вЂ” *The Mirror*
> *"99.9% reflectivity. Even plasma just bounces right off."*

A slow-moving gastropod from the **Silicon Caves of Planet Ser**. Its shell is composed of high-density, naturally polished silicon вЂ” so reflective it can redirect high-energy plasma beams with near-perfect efficiency.

During the war, Mirlugs were used as tactical redirection pads. In the game, **Mirror pieces** are modeled after their extraordinary shells.

1. **Host** creates a room в†’ a unique **6-character room code** is generated (e.g., `KRTV4Y`).

6. **Full room protection** вЂ” a second joiner attempting to connect to an active room receives a `ROOM_FULL` rejection.

- рџ”Ѓ **Room code collision retry** вЂ” if a room ID is already taken, the host auto-generates a new one (up to 3 retries).

### Key Design Decisions

- **Immutable State** вЂ” `GameState.js` is a pure reducer: `(board, action, player, context) в†’ nextState`. No mutation; every action returns a fresh state snapshot.
- **Laser Tracing** вЂ” `traceLaserBeam()` uses a visited-set keyed by `(r, c, dr, dc)` to prevent infinite mirror loops. Returns the full beam path for the renderer.
- **Bot as Sandbox Player** вЂ” The bot uses the same `applySandboxAction()` as a human. It's an AI that simply picks the action; no special code paths needed.
- **No Backend** вЂ” Zero game server. State sync happens over WebRTC DataChannels. PeerJS's signaling server is only used for the initial connection handshake.

## рџ› пёЏ Getting Started

### Prerequisites
- **Node.js** `>= 18`

### Installation

```bash
git clone https://github.com/denzven/Lazer-Showdown.git
cd Lazer-Showdown

```

### Development


> **Note:** HTTPS is enabled automatically via `vite-plugin-mkcert`. This is **required** for WebRTC to function. Your browser may display a certificate warning on first launch вЂ” this is expected for local development.

- `https://localhost:5173` вЂ” local machine

### Building for Production

```bash

npm run preview     # Preview the production build locally
```

### Deploying to GitHub Pages

```bash
npm run deploy
```

This runs `vite build` and then publishes the `dist/` folder via `gh-pages`.
